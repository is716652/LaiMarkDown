# 来MarkDown 2.0 — 业务需求说明书

> 本文档反映截至 2026-06-17 的真实实现状态（v2.0 基础 + v2.1 增强 + v2.2 file-association 与预览区跳转）。
> 代码版本：`package.json` 仍为 `2.0.0`（未 bump minor），主分支 commit `3b8356b`。

## 一、产品定位

**来MarkDown 2.0** 是一款面向高校教师、技术写作者及 Markdown 重度用户的**现代化桌面 Markdown 编辑器**。

- **形态**：Windows 桌面应用（Electron 32 + Vite 5 + React 18 + TypeScript 5）
- **包形态**：~98 MB NSIS 安装包；绿色版（`win-unpacked/`）可直接双击运行
- **目标场景**：日常写作、课程标准编写、文档维护、笔记整理
- **设计原则**：浅色主题默认、启动即用、无命令行/无开发依赖

## 二、目标用户

| 用户类型 | 使用场景 | 核心诉求 |
|----------|---------|---------|
| 高校教师 | 课程标准、讲义、试卷的 Markdown 编写 | 打字流畅、所见即所得、可导出 PDF/DOCX |
| 技术作者 | 博客、API 文档、教程撰写 | 代码高亮、数学公式、流程图支持 |
| Markdown 重度用户 | 个人笔记、文档归档 | 标签页、文件夹管理、主题切换、文件搜索 |

## 三、核心功能

### 3.1 文档编辑

#### 3.1.1 CodeMirror 6 编辑器
- **行号显示**（可开关）
- **自动换行**（可开关）
- **Tab Size**（2 / 4 / 8，默认 2）
- **字体设置**（字号、字体族；持久化到 `userData/settings.json`）
- **撤销/重做**（Cmd/Ctrl+Z / Cmd/Ctrl+Shift+Z）
- **多光标编辑**（Alt+点击、矩形选区）
- **查找/替换**（Cmd/Ctrl+F / Cmd/Ctrl+H）
- **Markdown 语法高亮**（基于 `@codemirror/lang-markdown`）
- **代码块折叠**（`foldGutter`）
- **行级滚动同步**（v2.1）：编辑器和预览通过 `marked.lexer` 算出的源行号锚点对齐，不再用 fraction

#### 3.1.2 多 Tab 编辑
- **新建 Tab**（Cmd/Ctrl+N）
- **Tab 切换**（点击 tab 标签）
- **Tab 关闭**（× 按钮；最后一个 Tab 不能关）
- **未保存标记**（标题前 ● 圆点）
- **已保存状态**（标题前 ✓ 标记，状态栏显示）

#### 3.1.3 内容状态
- **字/词/行统计**（状态栏右下角）
- **文件路径显示**（状态栏）
- **保存指示**（● 未保存 / ✓ 已保存）

#### 3.1.4 MD 语法提示工具栏（FormatToolbar，v2.1）
- **行首前缀类**：H1-H6 标题、引用、列表（有序/无序/任务）、分隔线
- **行内包裹类**：粗体（`**`）、斜体（`*`）、删除线（`~~`）、行内代码（`` ` ``）、链接（`[]()`）
- **数学公式**：行内 `$...$`、块级 `$$...$$` 模板
- **交互**：点击按钮自动包裹选区 / 在行首加前缀；tooltip 显示对应语法模板

### 3.2 文件操作

#### 3.2.1 打开文件
- **快捷键**：Cmd/Ctrl+O
- **菜单**：文件 → 打开文件…
- **支持格式**：`.md`、`.markdown`、`.txt`、所有文件
- **重复打开检测**：同一文件再次打开时，切到已有 Tab，不会新建

#### 3.2.2 打开文件夹（侧边栏工作区）
- **快捷键**：Cmd/Ctrl+Shift+O
- **菜单**：文件 → 打开文件夹…
- **功能**：
  - 文件夹树状展开
  - 子目录懒加载（点击展开时再读）
  - 点击 .md 文件直接打开到编辑器
  - 关闭文件夹（侧边栏 × 按钮）
- **遗留问题**：**侧边栏不排除 `node_modules` / `.git` 等巨型目录**（v2.1 漏了，剩余未实现）

#### 3.2.3 保存
- **保存**（Cmd/Ctrl+S）：已有路径直接写
- **另存为**（Cmd/Ctrl+Shift+S）：弹出保存对话框
- **未保存提示**（**剩余未实现**）：Tab 关闭前不弹确认框，dirty 状态只是视觉标记

#### 3.2.4 粘贴图片
- **触发**：编辑器内 Cmd/Ctrl+V 粘贴图片
- **行为**：
  - 自动保存到 `<md文件目录>/img/paste-yyyymmdd-HHMMSS-mmm.png`
  - 文档中插入 `![](./img/paste-xxx.png)` 相对路径引用
  - 未保存文件时存到 `%APPDATA%/lai-markdown-2/unsaved/img/`

#### 3.2.5 拖拽/URL 图片下载
- 支持拖入或粘贴图片 URL
- 自动下载到 `img/` 目录

#### 3.2.6 大文件流式打开（v2.1）
- **阈值**：> 1 MB 触发流式
- **行为**：
  - 先建空 Tab 占位
  - 状态栏显示加载进度（百分比 + loaded/total bytes）
  - 分批 256KB 插入 editor，每批 `await 0` 让主线程喘气
  - 完成后同步全文到 store
- **作用**：>5MB 大文件不再卡启动

#### 3.2.7 侧边栏文件名搜索（v2.1）
- **触发**：侧边栏顶部搜索框
- **算法**：主进程递归扫描（深度限制 6 层），自动排除 `node_modules` / `.git` / `dist` / `build` 等
- **交互**：200ms 防抖过滤，键盘上下/Enter 跳转
- **排序**：按相对路径，中文 `zh-Hans-CN` locale

#### 3.2.8 Windows file association 启动参数（v2.2）
- **NSIS 注册**：装时自动把 `.md` / `.markdown` / `.txt` 关联到 `来MarkDown`
- **首次冷启动**：资源管理器右键 → 打开方式 → 选来 MarkDown → 软件启动 + 文件自动加载
- **已运行时再打开**：single-instance lock 触发 `second-instance`，**老进程**接收第二个进程的 `argv` → 拉起窗口 + 新 Tab 打开
- **多文件选中**：资源管理器选中多个 .md → 打开方式 → 来 MarkDown → 多个 Tab 一次打开，焦点在最后一个
- **macOS 备用路径**：`app.on('open-file')` 同样走 `openFilesFromMain` 队列
- **argv 解析防御**：`pickFileFromArgv` 排除 `.exe` 自身 / flag 参数 / 不存在的路径，只保留合法 `.md/.markdown/.txt` 绝对路径

### 3.3 预览渲染

#### 3.3.1 Markdown 解析
- **引擎**：`marked` 11（GFM 模式）
- **支持语法**：
  - 标题、段落、列表（有序/无序/任务列表）
  - 引用块
  - 表格
  - 行内/块级代码
  - 链接、图片
  - 分隔线
  - 加粗、斜体、删除线

#### 3.3.2 代码高亮（Prism 异步）
- **架构**：Web Worker 异步高亮，**不阻塞主线程**
- **支持语言**：markup、clike、javascript、typescript、js/ts、css、json、bash、python、java、c/cpp/csharp、go、rust、sql、yaml、markdown 等 20+ 种
- **代码块功能**：
  - 一键复制按钮
  - 语言标识
  - 等宽字体

#### 3.3.3 数学公式（KaTeX）
- **行内公式**：`$E = mc^2$`
- **块级公式**：
  ```
  $$
  \int_0^\infty e^{-x^2} dx = \frac{\sqrt{\pi}}{2}
  $$
  ```
- **错误降级**：公式错误时显示错误信息，不崩溃

#### 3.3.4 流程图（Mermaid）
- **触发**：代码块语言为 `mermaid`
- **特性**：
  - 懒加载（IntersectionObserver 触发）
  - 滚动到视野内才渲染
  - 错误时显示降级提示
- **示例**：
  ````
  ```mermaid
  graph TD
    A[开始] --> B{判断}
    B -->|是| C[执行]
    B -->|否| D[结束]
  ```
  ````

#### 3.3.5 防抖
- **预览渲染延迟**：150ms（用户连续打字时合并渲染）
- **作用**：避免每个字符触发全量重渲染

#### 3.3.6 预览区点击跳转 editor（v2.2）
- **机制**：每个 top-level block 注入 `data-source-line` + `data-source-span`（v2.2 扩展）
- **单击**：跳转 editor 对应行，光标到该行
- **双击**：跳转 + 选中整段（`span` 行数），按 Delete 直接删
- **视觉提示**：hover 段落/标题/列表/表格/代码块时淡紫高亮 + cursor: pointer
- **不拦截**：链接、代码块（pre/code）、图片、任务列表复选框、Mermaid 渲染区 → 走原本行为
- **快捷路径**：处于 preview-only 视图时点击预览区，自动切到 split-h 露出 editor

### 3.4 视图布局

#### 3.4.1 视图模式（4 种）
- **仅编辑**：编辑区撑满
- **仅预览**：预览区撑满
- **左右分栏**（split-h）：左右布局，可拖拽分栏
- **上下分栏**（split-v）：上下布局，可拖拽分栏

#### 3.4.2 分栏拖拽
- **触发**：鼠标按住中间分隔条
- **行为**：
  - 拖拽时实时调整比例（CSS 变量更新，不触发 React 重渲染）
  - 松手时保存比例到 settings store
  - 比例范围：15% ~ 85%
  - 切换分栏方向保留比例

#### 3.4.3 互换编辑/预览
- **快捷键**：Cmd/Ctrl+Alt+S
- **菜单**：视图 → 互换编辑/预览
- **行为**：左右/上下位置对调（CSS `flex-direction: row-reverse/column-reverse`）

#### 3.4.4 侧边栏切换
- **快捷键**：Cmd/Ctrl+B
- **菜单**：视图 → 切换侧边栏
- **行为**：打开/折叠左侧文件夹树

### 3.5 主题

#### 3.5.1 三种主题
- **浅色（light）**：默认主题，米白底 + 琥珀色强调
- **深色（dark）**：暗夜底 + 琥珀色强调
- **跟随系统（system）**：自动响应 `prefers-color-scheme`

#### 3.5.2 主题切换与持久化（v2.1）
- **快捷键**：菜单 → 视图 → 切换主题（循环 light → dark → system）
- **命令面板**：Cmd/Ctrl+Shift+P → 输入"主题"
- **首次启动**：默认浅色（CSS `:root` 默认值，无首屏闪烁）
- **存储**：v2.1 起持久化到 `userData/settings.json`（500ms debounce 写盘）

#### 3.5.3 设计令牌
- 浅色：`--bg: #fafafa`，`--text: #18181b`，`--accent: #f59e0b`
- 深色：`--bg: #1a1b26`，`--text: #c0caf5`，`--accent: #e0af68`
- 切换机制：CSS `[data-theme="dark"]` 属性选择器（**不是** class，避免首屏闪烁）

### 3.6 导出

#### 3.6.1 导出 PDF
- **触发**：工具栏 → 导出 → PDF / 命令面板
- **架构**：主进程 `webContents.printToPDF`（Electron 原生排版引擎）
- **优势**：
  - 中文表格 100% 正常（无 html2pdf.js 的兼容问题）
  - 不弹任何浏览器对话框
  - 一步到位
- **页面设置**：A4 + 14mm/12mm 边距 + 背景色

#### 3.6.2 导出 HTML
- **触发**：工具栏 → 导出 → HTML / 命令面板
- **产物**：独立 HTML 文件，内嵌完整 CSS（亮/暗双主题适配）
- **特点**：可分发、可邮件发送、双击直接打开

#### 3.6.3 导出 DOCX（v2.1）
- **触发**：工具栏 → 导出 → DOCX / 命令面板
- **架构**：主进程 `markdownToDocxBuffer`（`docx@8.5.0` 包）
- **支持**：标题、段落、列表（有序/无序/任务）、引用、代码块、表格、分隔线；行内格式（粗体/斜体/删除线/链接/行内代码）；数学公式降级为纯文本
- **交互**：原生保存对话框，文件名默认 `untitled.docx`

### 3.7 命令面板

#### 3.7.1 打开
- **快捷键**：Cmd/Ctrl+Shift+P
- **菜单**：视图 → 命令面板
- **工具栏**：右上角命令面板按钮

#### 3.7.2 命令列表
- **文件**：新建、打开文件、打开文件夹、保存
- **视图**：仅编辑、仅预览、左右/上下分栏、互换、侧边栏、主题
- **主题**：浅色/深色/跟随系统
- **设置**：增大/减小字号、切换行号、切换换行
- **导出**：PDF、HTML、DOCX

#### 3.7.3 交互
- 模糊搜索（label / description / group）
- 上下方向键选择
- Enter 执行
- Esc 关闭

### 3.8 窗口与界面

#### 3.8.1 窗口控制
- **自定义标题栏**（无 frame / hidden titleBar）
- **最小化 / 最大化 / 关闭**（标题栏右侧三个按钮）
- **背景色**：浅色 `#fafafa`（与 CSS 默认主题一致，避免首屏闪烁）
- **初始尺寸**：1400 × 900，最小 960 × 600

#### 3.8.2 状态栏
- 文件路径、未保存/已保存、行/词/字符统计、viewMode 指示、大文件加载进度条（v2.1）

#### 3.8.3 标签栏
- 当前打开的所有 tab
- 未保存用 ● 标记
- 关闭按钮（×）
- 新建 tab 按钮（+）

## 五、非功能需求

### 5.1 性能
- 启动时间（双击到窗口出现）：< 2 秒
- 预览渲染延迟：150ms 防抖
- 代码高亮：Web Worker 异步，主线程不卡
- Mermaid 渲染：IntersectionObserver 懒加载
- 大文件（>1MB）：分块 256KB 异步插入

### 5.2 兼容性
- Windows 10 / 11（x64）
- 屏幕分辨率：1280×720 及以上

### 5.3 资源占用
- 安装包大小：~98 MB（含 electron runtime）
- 内存占用：空闲 < 200 MB
- 磁盘占用：安装后 ~250 MB

### 5.4 安全
- `contextIsolation: true`（renderer 与 preload 隔离）
- `nodeIntegration: false`
- 自定义 `safe-file://` 协议（本地图片沙箱访问）
- CSP 头限制

### 5.5 可靠性
- **单实例锁**（v2.0 基础 + v2.2 增强：second-instance 透传 argv）
- **窗口关闭时退出主进程**
- 进程自清理（关窗口 = 进程全清，无残留）

### 5.6 持久化
- 设置（主题、字体、字号、tabSize、wordWrap、行号、预览字体、分栏比例）→ `userData/settings.json`
- 500ms debounce 写盘
- 字段白名单 + 类型校验

## 六、版本与发布

### 6.1 版本历史
- **v2.0.0**（2026-06 之前）—— 重写自 1.0，浅色默认，完整中文支持

> 注：`package.json` 的 `version` 字段未跟随 commit bump，停留在 `2.0.0`，发布时建议 bump 到 `2.2.0`。

### 6.2 当前未实现（剩余 backlog）
- **侧边栏排除巨型目录**（`node_modules` / `.git` / `dist` / `build`）—— v2.1 漏了
- **关闭 Tab 前未保存确认** —— dirty 状态只视觉，不阻断
- **大文件（>5MB）编辑流畅度优化** —— v2.1 流式打开是 load 阶段，编辑阶段仍按全量 codemirror 处理
- **global 全局搜索**（v2.1 计划文档提到，但只做了"侧边栏文件名搜索"）
- **macOS 版本** —— 代码有 `app.on('open-file')` 占位，但**没**做 macOS 打包

### 6.3 下一步建议
1. bump `package.json` version 到 `2.2.0`
2. 修侧边栏 `node_modules` 排除（成本：~30 行）
3. macOS 打包（electron-builder 加 `mac.target: 'dmg'`）
4. Tab 关闭未保存提示（成本：~20 行）
5. 全局搜索（成本：~2 天，需要建索引）

## 九、验收标准

| 项 | 标准 |
|----|------|
| 安装 | 双击 NSIS 安装包，按向导完成安装；.md / .markdown / .txt 自动关联来 MarkDown（v2.2） |
| 启动 | 双击桌面"来MarkDown"图标，2 秒内出窗口，浅色主题 |
| 编辑 | 打开/新建/保存 .md 文件正常，光标移动流畅 |
| **Preview 点击跳转**（v2.2）| split 模式下，鼠标 hover 任意 block 看到淡紫高亮；单击跳到 editor 对应行；双击跳 + 选中整段 |
| **file association 启动**（v2.2）| 资源管理器右键 .md → 打开方式 → 选来 MarkDown → 软件起来 + 文件自动加载；多文件打开多个 tab；已运行时右键 .md 拉起窗口 + 新 tab |
| 预览 | 编辑内容实时预览（150ms 防抖），代码块有高亮 |
| 滚动同步 | split 模式下编辑器和预览按**行级锚点**对齐（v2.1）；用户主动滚 preview 时编辑器只跟滚视图、光标不跳（v2.2 修复） |
| 分栏 | 左右/上下分栏可拖拽，比例持久 |
| 主题 | 切到深色正常，再切回浅色正常；**重启后主题保留**（v2.1 持久化） |
| 导出 PDF | 弹出保存对话框，导出后中文表格正常 |
| 导出 DOCX（v2.1）| 弹出保存对话框，导出 .docx 可在 Word 打开，标题/段落/列表/表格/代码块正常 |
| 大文件（v2.1）| 打开 > 1MB .md，状态栏显示加载进度，编辑不卡 |
| 侧边栏搜索（v2.1）| 侧边栏顶部输入框 200ms 防抖过滤，上下/Enter 跳转 |
| 进程清理 | 关闭窗口后任务管理器无残留进程 |
| 卸载 | 控制面板卸载干净，无残留 |
